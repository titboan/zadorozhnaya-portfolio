---
name: svg-fill-no-css-vars-in-img
description: SVG загруженный как <img> не видит CSS-переменные страницы — использовать hardcoded значения
type: feedback
---

SVG-файл, загруженный через `<img src="logo.svg">`, рендерится в изолированном контексте. CSS-переменные родительской страницы (`:root { --color }`) в него не передаются.

**Почему:** `fill="var(--fill-0, white)"` → браузер не находит `--fill-0` → fallback `white` должен работать, но на практике часть браузеров рендерит fill как `none` или `black` → логотип невидим на тёмном фоне.

**Как применять:** В SVG-файлах, используемых как `<img>`, всегда использовать hardcoded цвета:

```svg
fill="white"  <!-- вместо fill="var(--fill-0, white)" -->
```

Если нужна смена цвета — инлайнить SVG в JSX или использовать CSS `filter: brightness()`.
