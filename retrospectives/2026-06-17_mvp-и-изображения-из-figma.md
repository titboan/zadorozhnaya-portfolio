# Рефлексия: MVP + реальные изображения из Figma

Дата: 2026-06-17

---

## 1. Какая задача была

Две задачи за одну сессию:
- **Часть 1 (предыдущая сессия):** создать портфолио-сайт с нуля по дизайну из Figma — React + Vite, 4 секции (Hero, Services, Works, Contacts), адаптивность, GitHub
- **Часть 2 (эта сессия):** заменить цветные заглушки в Works и пустой Hero реальными изображениями из Figma

---

## 2. Как решали

**Часть 1 (MVP):**
- Читали Figma через MCP — `get_design_context` для каждой секции, скриншоты для визуальной сверки
- Создали инфраструктуру до кода: CLAUDE.md, ROADMAP, plans, skills
- Написали компоненты вручную (Vite-шаблон не сработал — папка уже существовала)
- Проблема с Hero фото: использовали `import` вместо строкового пути — Vite пытался обработать как ES-модуль, файл не нашёл. Исправили на `src="/images/hero-photo.jpg"`

**Часть 2 (изображения):**
- `get_design_context` для node 0:44 (Главная) вернул все URL изображений
- `curl` скачал 7 файлов в `public/images/`
- `work-3-poly.jpg` весил 16 МБ → перескачали через `download_assets` с `defaultScale: 0.5` → 50 КБ
- Works.jsx: заменили поле `color` на `image`, `<div style={{bg}}>` на `<img>`

---

## 3. Решили — да / нет / частично

**Да.** MVP полностью работает: все 4 секции, адаптивность, реальные изображения, GitHub.

---

## 4. Что можно было лучше

- **nodeId 0:613 оказался не главным лендингом**, а страницей кейса "Солнечный берег" — пришлось искать правильный node (0:44). Стоит в начале сессии вызывать `get_metadata` для ориентации по структуре файла.
- **16 МБ изображение** — нужно проверять вес файла сразу после скачивания и оптимизировать до коммита, а не после обнаружения.
- **Инструментов для сжатия нет** на машине пользователя — PIL и ImageMagick отсутствуют. Решение через `download_assets` с `defaultScale` — хорошая альтернатива, стоит запомнить.

---

## 5. Что узнал нового о проекте или пользователе

- Борис учится через создание реального проекта — важно объяснять каждый шаг (что / зачем / даёт)
- Figma-файл содержит не только дизайн сайта, но и контент кейсов (страницы работ для портфолио) — нужно различать их при навигации по nodeId
- Vite трактует пути начинающиеся с `/` из `public/` как статику, а `import` — как ES-модуль через бандлер; картинки из `public/` всегда строкой, не импортом
