# Ретроспектива: экспорт изображений из Figma

## Задача
Скачать все изображения кейсов из Figma-файла `rA5AGt3zuM05YM7lzS0CoC` для последующей вёрстки страниц проектов.

## Как решали
1. Через Figma MCP получили метаданные файла — структура оказалась слишком большой (>115к символов), сохранилась во временный файл
2. Разобрали структуру через Python: нашли 6 кейсов, 2 коллекции изображений (АГРОЦЕНТР 29 шт, Провыходной 20 шт), монтажные области и крупные фреймы
3. Добавили Figma Personal Access Token в `.env`, создали папки `public/images/cases/`
4. Первый батч (АГРОЦЕНТР, 29 файлов) — скачался успешно через REST Images API
5. Второй батч (Провыходной) — Figma ответил 429 Too Many Requests
6. Все последующие попытки — 429, `Retry-After: 343716` (~4 дня)
7. Попытка через Figma MCP `get_screenshot` — исчерпан лимит Starter плана

## Решили — частично
- ✅ АГРОЦЕНТР: 29 изображений скачаны в `public/images/cases/agrocentr/`
- ❌ Провыходной: 20 изображений — не скачаны
- ❌ Монтажные области: 8 изображений — не скачаны
- ❌ Превью фреймов (Солнечный берег, Айдентика, Логотипы, Главная) — не скачаны

## Что можно было лучше
- Не качать все 29 файлов одним пакетным запросом — разбить на батчи по 5-10 с паузами между ними
- Сначала проверить rate limit через один тестовый запрос
- Узнать заранее что бесплатный MCP тоже имеет лимит на скриншоты

## Что узнал нового
- Figma REST Images API: после пакетного экспорта банит на несколько дней (`Retry-After: 343716s`)
- Figma MCP Starter plan: ограниченное число вызовов `get_screenshot`
- Безопасный темп: не более 5-10 экспортов за раз с паузами 10-15 секунд
